# gdo-web

Client-side Next.js + TypeScript app for the [Gradient Descent and Optimizers](../README.md)
project. All optimizer/scheduler/landscape math runs **in the browser** — no backend.

The math in `lib/` is a TypeScript port of the Python `gdo` package. A Vitest parity suite
(`tests/`) asserts it matches the Python reference — regenerated by `python ../scripts/dump_reference.py`
into `tests/fixtures/reference.json` — to six decimals.

```bash
npm install
npm run dev        # http://localhost:3000
npm test           # parity suite
npm run typecheck  # tsc --noEmit
npm run lint       # eslint
npm run build      # production build (Vercel)
```

## Structure

- `lib/` — `surfaces.ts`, `optimizers.ts`, `schedulers.ts`, `mlp.ts` (the ported math)
- `components/` — `LossSurfaceCanvas` (heatmap + trajectories), `LineChart`, `tabs/`
- `app/` — Next.js App Router shell (dark instrument theme)
- `tests/` — Vitest parity tests + fixtures

## Deploy

Vercel project with root directory `web/`, framework preset Next.js. No environment variables.
